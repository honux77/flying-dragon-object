# 006 — 인게임 한글화

## 경과

1. **메뉴 UI 한글화** — Galmuri9(OFL-1.1) BDF를 `tools/mkfont.py`로 KS 완성형
   2350자 10x12 비트맵 헤더(`font_kr.h`)로 변환, menu.c의 `draw_str`을
   UTF-8 인식으로 확장 (ASCII 8x8 + 한글 10x12 혼합 렌더링).

2. **wbmljb 부틀렉 셋 지원** — 부틀렉 프로그램 롬(wbml.01, m-6.bin, m-7.bin)은
   MC-8123을 미리 복호화한 `<32KB opcode 뷰> + <32KB data 뷰>` 형식.
   machine.c가 `wbml.01` 존재 시 자동으로 부틀렉 경로 로드 (복호화 생략).

3. **한국어 패치 적용** — `tools/mkbootleg.py`가 `roms-kr/` 생성:
   - 실제 wbmljb 롬(roms-wbmljb/) 우선, 없으면 wbml 셋을 복호화해 생성
   - CRC32 검증 (wbml.01=66482638, m-6=8c08cd11, m-7=11881703)
   - 한국어 패치 IPS 적용 (제작: 주니, korean-patch/wbmljb/)
   - `./build/flydo roms-kr`로 실행 → 인게임 한글 동작 확인

## 문자열 인코딩 분석 (리버싱 결과)

- 대사 문자열: m-6.bin 데이터 뷰 `0x8F1B–0x9EA5`, `0xFF` 종결, 140개
- 청크 선두 ~3바이트는 제어 코드(창 종류/좌표), 중간 제어(0xFE 등) 존재
- 바이트 `b` → fg 타일 `0x800 + b`
- 원판(일본어)은 1바이트 = 가나 1자. 탁점은 합성 프리픽스 사용
- 한국어 패치의 음절 인코딩:
  - **완성 음절**: 1바이트 (타일 0x840 이후에 통글자)
  - **희귀 음절**: 2바이트 = 초성 타일(0x00–0x0D = ㄱ~ㅎ) + 몸통 타일(중성+종성)
    → 화면에서 가로 2칸 차지 (가독성 저하의 원인)
- 타일 패치는 3개 타일롬의 0x4000–0x47E8 (= 타일 0x800–0x8FC, 214개 변경)
- 글자 펜: 7, 배경 펜: 1

## 다음 단계: 에뮬레이터 측 Galmuri9 렌더링

MAME과 달리 우리는 렌더러를 소유하므로 타일 그래픽 패치가 불필요:

1. `tools/krdump.py`로 전체 대사를 라벨된 시트로 렌더 (build/krmap/*.png)
2. 시트를 판독해 "타일 코드 → 음절" 매핑 테이블 작성
   - 초성 14 + 몸통 ~95 + 완성 ~150
   - 몸통은 (중성, 종성)만 알면 유니코드 조합식으로 자동 합성
3. video.c 또는 main.c에서 fg 타일맵 스캔:
   - 한글 타일 코드 감지 → 해당 셀 위에 Galmuri9(10x12) 오버레이
   - 2칸 조합 페어는 16px 공간에 한 글자 — 통합 렌더링으로 가독성 회복
4. 데모 대사("골드를 얻었다")로 검증

## 구현 완료 (v1)

### 타일→음절 매핑 도출 (분업)

- `tools/krdump.py` — 일본어 원문(원본 타일)과 한국어(패치 타일)를
  같은 번호로 나란히, 셀마다 16진 라벨을 붙여 시트 23장 생성
- 사용자가 140개 문자열 전체를 받아쓰기 (`kr_strings_template.txt`)
- `tools/kralign.py` — 받아쓰기와 바이트열 자동 정렬:
  - 조합 페어는 초성 일치 검증, 교차 투표로 매핑 수렴
  - 미지 타일은 디코드 결과의 문맥 + 타일 직접 렌더로 판독
  - 수동 확정은 `tools/kr_overrides.txt`에 기록 (자동 투표보다 우선)
- 함정: 대사창 테두리 타일(0x8D4-0x8DB)을 음절로 오판독하면
  화면에서 테두리가 글자로 덮임 — 매핑 금지 처리

### 오버레이 렌더러 (krtext.c)

- fg 타일맵(page 0) 스캔, 텍스트 타일은 코드 0x800-0x8FF만
  (`(code & 0xf00) == 0x800` — 0xDA0 같은 그래픽 타일 오인 주의)
- 초성 타일 + 몸통 타일 페어 → 유니코드 조합식으로 음절 합성,
  2칸(16px)에 Galmuri9 한 글자 중앙 정렬
- 모음 연결([무][ㅓ]→뭐) 합성 지원
- main.c: 패치 타일 시그니처(타일 0x800 = ㄱ 글리프) 감지 시 활성화

### 번역 재인코딩 (krencode.py)

받아쓰기 파일이 곧 게임 대사가 되도록 m-6.bin 문자열을 재작성:

- 문자열 구조: `제어(3B) [텍스트] (7B xx=줄바꿈 | FE+제어3B=새 창) ... FF`
  — 분리자/제어는 그대로 보존해 창 레이아웃 유지, 바이트 길이 불변
- 오버레이가 글꼴을 그리므로 타일 코드 재배치가 자유로움:
  - 빈도순 상위 음절 → 1바이트 단독 코드 (~115개)
  - 희귀 음절 → 초성 코드 + 몸통 코드 2바이트 (몸통 ~88개)
  - 숫자/구두점/커서/테두리 코드는 예약 (원본 타일 그대로)
- `krtext.h`를 새 인코딩 기준으로 생성 → 인코더와 오버레이가 항상 1:1
- 창 너비를 넘는 번역은 잘림 경고 출력 → 템플릿에서 줄여야 함

### 빌드 절차

```sh
python3 tools/mkbootleg.py   # 부틀렉 롬 + IPS → roms-kr/
python3 tools/krencode.py    # 번역 재인코딩 + krtext.h 생성
./build.sh
./build/flydo roms-kr
```
