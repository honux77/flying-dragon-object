# 006 — 인게임 한글화

## 경과

1. **메뉴 UI 한글화** — Galmuri9(OFL-1.1) BDF를 `tools/mkfont.py`로 KS 완성형
   2350자 10x12 비트맵 헤더(`font_kr.h`)로 변환, menu.c의 `draw_str`을
   UTF-8 인식으로 확장 (ASCII 8x8 + 한글 10x12 혼합 렌더링).

2. **wbmljb 부틀렉 셋 지원** — 부틀렉 프로그램 롬(wbml.01, m-6.bin, m-7.bin)은
   MC-8123을 미리 복호화한 `<32KB opcode 뷰> + <32KB data 뷰>` 형식.
   machine.c가 `wbml.01` 존재 시 자동으로 부틀렉 경로 로드 (복호화 생략).

3. **한국어 패치 적용** — `tools/mkbootleg.py`가 `roms-kr/` 생성:
   - 실제 wbmljb 롬(roms-wbmljb/) 우선, 없으면 wbml 셋을 복호화해 생성
   - CRC32 검증 (wbml.01=66482638, m-6=8c08cd11, m-7=11881703)
   - 한국어 패치 IPS 적용 (제작: 주니, korean-patch/wbmljb/)
   - `./build/flydo roms-kr`로 실행 → 인게임 한글 동작 확인

## 문자열 인코딩 분석 (리버싱 결과)

- 대사 문자열: m-6.bin 데이터 뷰 `0x8F1B–0x9EA5`, `0xFF` 종결, 140개
- 청크 선두 ~3바이트는 제어 코드(창 종류/좌표), 중간 제어(0xFE 등) 존재
- 바이트 `b` → fg 타일 `0x800 + b`
- 원판(일본어)은 1바이트 = 가나 1자. 탁점은 합성 프리픽스 사용
- 한국어 패치의 음절 인코딩:
  - **완성 음절**: 1바이트 (타일 0x840 이후에 통글자)
  - **희귀 음절**: 2바이트 = 초성 타일(0x00–0x0D = ㄱ~ㅎ) + 몸통 타일(중성+종성)
    → 화면에서 가로 2칸 차지 (가독성 저하의 원인)
- 타일 패치는 3개 타일롬의 0x4000–0x47E8 (= 타일 0x800–0x8FC, 214개 변경)
- 글자 펜: 7, 배경 펜: 1
