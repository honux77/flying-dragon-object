## 하드웨어 구조 요약

wbml은 Sega **System 2** 기판에서 동작한다.

```
MASTER_CLOCK = 20 MHz
                │
        ┌───────┴────────┐
    ÷5  │                │
   Main Z80             ÷5
   @ 4 MHz         Sound Z80
   (MC-8123 암호화)    @ 4 MHz (SOUND_XTAL/2)
        │                │
   8255 PPI ──── 사운드 래치 ──── SN76489A × 2
        │
   System 2 비디오 (512×224)
```

### CPU: NEC MC-8123

일반 Z80이 아니라 NEC가 만든 암호화 버전이다.  
롬에 저장된 바이트는 암호화되어 있고, 실제 실행 시 칩 내부에서 복호화된다.  
복호화 방식: 칩에 내장된 2KB 키 데이터(`317-0043.key`)를 사용해  
**opcode 뷰**(M1 사이클에서 읽히는 명령어 바이트)와 **data 뷰**(데이터로 읽히는 바이트)를 분리 생성.

에뮬레이터에서는 `mc8123_decode()`가 롬 로드 시 한 번에 두 뷰를 미리 계산해 둔다.
Z80 코어는 `fetch_opcode` 훅을 통해 opcode 뷰를, `read_byte`를 통해 data 뷰를 읽는다.

### 비디오: System 2 렌더러

- 타일맵: 256×256 픽셀 페이지 8개, 각 타일 8×8
- 스프라이트: 최대 32개, 8×8 또는 8×16
- 컬러: PROM 기반 룩업 팔레트 (256색)
- 충돌 감지: 믹서/스프라이트 충돌 레지스터를 하드웨어가 자동 세트

### 사운드: SN76489A PSG × 2

- SN1: SOUND_XTAL/4 = 2 MHz 클럭, gain 0.40
- SN2: SOUND_XTAL/2 = 4 MHz 클럭, gain 0.60
- 각 칩: 3개 톤 채널 + 1개 노이즈 채널

## 구현 방법

| 파일 | 역할 |
|------|------|
| `mc8123.c` | MAME `mc8123.cpp` 포팅. 키를 이용한 opcode/data 분리 복호화 |
| `z80/z80.c` | superzazu/z80 MIT 라이선스 코어. fetch_opcode 훅 추가 |
| `machine.c` | 메모리맵, 뱅킹, PPI 8255, 메인/사운드 Z80 인터리브 실행 |
| `video.c` | MAME `system1_v.cpp` 포팅. 타일맵 8페이지 + 스프라이트 32개 렌더링 |
| `main.c` | SDL2 윈도우, 입력, 60Hz 루프 |
| `dbg.c` | 헤드리스(창 없이) 진단 도구 |

### 프레임 루프 구조

```
machine_run_frame() {
    // 메인 Z80: 프레임당 CYC_PER_FRAME (= 4,000,000 / 60.06 ≈ 66,581) 사이클 실행
    //   VBLANK 인터럽트 (RST 38h) 발생
    // 사운드 Z80: 프레임당 SCYC_PER_FRAME (= 4,000,000 / 60.06) 사이클 실행
    //   스캔라인 32마다 IRQ 4회 (= 프레임당 4 IRQ)
    video_render()
}
```
