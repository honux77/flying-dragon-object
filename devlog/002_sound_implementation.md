# 002 — 사운드 구현 (버그 있는 초기 버전)

**날짜**: 2026-06-11  
**커밋**: `958a028` 기본 구현 완료. 소리가 이상한 문제가 있음

---

## 무엇을 추가했나

- `sn76489.c/h`: SN76489A PSG 에뮬레이션 직접 구현
  - 3채널 톤 생성기 (주파수 카운터 + 토글)
  - 1채널 노이즈 생성기 (LFSR 15비트)
  - 샘플레이트 리샘플링 (44.1kHz)
- `machine.c`: 사운드 CPU 추가, PPI 래치/NMI 연결, SN 칩 I/O 매핑
- `main.c`: SDL2 오디오 장치 열기, 프레임당 PCM 큐잉
- `dbg.c`: 사운드 진단 카운터 (NMI 횟수, IRQ 횟수, SN 쓰기 횟수)

---

## 사운드 CPU와 메인 CPU의 통신 구조

```
메인 CPU                          사운드 CPU
─────────────────────────────────────────────
OUT 0x14, 커맨드값
   │
   ▼
[사운드 래치 (soundlatch)]
   │
   ▼ NMI 발생
   │
   └──► NMI 핸들러 @ 0x0066
            LD A, (E000h)   ; 래치 읽기
            ; 커맨드에 따라 BGM/SFX 큐에 넣기
            RETN            ; 인터럽트 복귀
   ▼
주 루프 @ 0x00E0
   └──► 4회/프레임 IRQ (스캔라인 32 주기)
            sound driver 처리
            SN76489A에 음정/볼륨 쓰기
```

## 이 시점의 버그 증상

- 부팅 음("피로롱"): 단순한 단음으로 늘어져서 재생됨
- 코인 투입 효과음: 짧아야 하는데 길게 끌림
- MAME 기준 커맨드 0x17 수신 후 바로 나와야 할 SN2 쓰기 시퀀스가 **약 52프레임(0.87초) 지연**됨
